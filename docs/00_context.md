# Context

Проект: Проектный радар

Коротко: инструмент для руководителя проекта, который по набору измерений (16 осей) строит профиль проекта, рекомендует класс методологии управления и набор обязательных governance-контролей, а также отслеживает динамику профиля во времени и сигнализирует, когда подход пора менять.

1) Зачем мы это делаем

В реальных проектах методологию выбирают “по привычке” (waterfall vs agile), а затем не пересматривают, даже когда меняются требования, риски, регуляторика, команда, интеграции, срок/бюджет. Это приводит к поздним сюрпризам: срывы сроков, перерасход, дефекты качества, конфликт стейкхолдеров.

Проектный радар решает 3 задачи:

Диагностика: быстро измерить проект по ключевым осям.

Рекомендация: подобрать подходящий класс модели управления (или гибрид).

Динамика: видеть тренды изменений профиля за 3–6 weekly-ревью и получать сигналы “пора пересмотреть подход”.

2) Что является результатом (MVP)

MVP — это работающий прототип, который позволяет:

2.1 Ввод данных

создать “карточку проекта”

выставить оценки по 16 осям (шкала 1–5) с подсказками уровней

(опционально) добавить weekly-статус текстом/транскриптом

2.2 Авто-вычисления

строит радар-диаграмму на 16 осей

формирует компактный код профиля:
S# C# K# F# | T# B# + флаги R/Q

рекомендует:

класс методологии (Stage-Gate/Waterfall, Iterative, Incremental/Agile, Spiral/Risk-driven, Hybrid)

обязательные контроли (must-controls) под профиль

сигналы (triggers) при изменении профиля во времени

2.3 История и динамика

хранит историю ревью (минимум 6 последних)

показывает тренды по осям и агрегатам

сигнализирует о пересечении порогов / конфликтах осей
